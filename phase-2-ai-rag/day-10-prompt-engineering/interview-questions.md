# Day 10 — Prompt Engineering: Interview Questions

1. How do you prevent prompt injection in a system that processes untrusted user text?
2. When would you choose few-shot over zero-shot prompting?
3. Why enforce structured JSON output for agent-to-agent communication instead of free text?
4. How do you manage prompts at scale across many agents in production?
5. What's the tradeoff of chain-of-thought prompting in a latency-sensitive system?
6. What's the difference between a system prompt and a user prompt, and why does that split matter in a multi-tenant platform like AESP?
7. How would you A/B test two versions of an agent's system prompt safely in production?
8. What temperature would you set for a classification task vs. a customer-facing draft reply, and why?
9. How do you defend against a model claiming it followed instructions (e.g., "I did not leak PII") when you can't fully trust its self-report?
10. Describe a regression testing strategy for prompts, analogous to unit tests for code.
